# US-0004

## Title

Implement Soft Delete for Knowledge Object

---

## Goal

Complete the Knowledge Object lifecycle by introducing Soft Delete.

A deleted Knowledge Object shall remain stored in the repository for history and audit purposes but shall be considered logically deleted.

Physical deletion is out of scope.

---

## Requirements

Implement:

- DeleteKnowledgeObjectCommand
- DeleteKnowledgeObjectHandler

Extend:

- KnowledgeObjectService

Reuse:

- Repository `update()` method

Do not introduce a repository `delete()` method.

---

## Domain Rules

Soft Delete changes the Knowledge Object status to:

`DELETED`

The operation shall:

- create a new version;
- preserve all previous history;
- emit `KnowledgeObjectDeleted` Domain Event.

Deleting an already deleted object shall raise:

`KnowledgeObjectAlreadyDeleted`

A deleted object cannot be:

- updated;
- archived;
- restored.

These operations shall raise:

`InvalidKnowledgeObjectStateTransition`

---

## Domain Events

Create:

`KnowledgeObjectDeleted`

It shall inherit from `BaseDomainEvent`.

Handlers must not publish the event.

---

## Repository

No new repository methods.

Persist the modified aggregate using:

`update()`

---

## Tests

Add integration tests covering:

Create

↓

Delete

↓

Status == DELETED

↓

Version incremented

↓

History preserved

Verify:

- deleting twice raises `KnowledgeObjectAlreadyDeleted`;
- update after delete fails;
- archive after delete fails;
- restore after delete fails;
- existing lifecycle behavior remains unchanged.

---

## Acceptance Criteria

- Soft Delete implemented.
- Repository API unchanged.
- Version history preserved.
- Domain Event created.
- Domain exceptions used.
- Existing tests pass.
- New tests pass.
- Ruff passes.
- Import validation passes.

---

## Non-Goals

- Physical deletion.
- Restore from deleted state.
- Search filtering.
- Database implementation.
