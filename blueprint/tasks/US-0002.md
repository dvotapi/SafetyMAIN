# US-0002

## Title

Update Knowledge Object with Versioning

---

## Goal

Implement immutable updates for Knowledge Objects.

Every successful update creates a new version.

Historical versions remain available.

---

## Requirements

Implement:

UpdateKnowledgeObjectCommand

UpdateKnowledgeObjectHandler

Version increment

Version history in memory

---

## Rules

KnowledgeObjectHeader.id never changes.

organization_id never changes.

object_type never changes.

Only payload and status may change.

Every update increments version.

---

## Repository

Extend InMemoryRepository.

Maintain object history.

Provide:

get()

update()

history()

---

## Tests

Verify:

Create object

↓

Update

↓

Version == 2

↓

History contains Version 1

↓

Current object contains Version 2

---

## Acceptance Criteria

✓ Immutable update

✓ Version history stored

✓ Tests pass

✓ Previous versions preserved
