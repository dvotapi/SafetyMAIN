# US-0007

## Title

Search Knowledge Objects

## Goal

Introduce deterministic in-memory search for Knowledge Objects.

The search capability shall allow Application Layer queries to find Knowledge Objects within an organization using structured filters.

This task must not introduce full-text search, database-specific behavior, or external search engines.

## Acceptance Criteria

- Structured Knowledge Object search implemented.
- Organization boundary always enforced.
- Type, status and exact metadata filters supported.
- Deleted objects excluded by default.
- Deleted objects can be explicitly requested.
- JSON metadata matching is type-sensitive.
- Results are deterministic.
- Pagination and total count implemented.
- Search results are immutable.
- Read handler does not commit.
- Repository contract remains infrastructure-independent.
- All tests pass.
- Ruff passes.
- Import validation passes.
- Architecture tests pass.
