# TASK-0014

## Title

Implement the first Knowledge Object Domain Model

---

## Goal

Implement the minimal immutable domain model for the SafetyMAIN Core.

This is the first implementation of the Knowledge Object Specification.

The implementation must strictly follow the approved architecture.

---

## Context

Knowledge Objects are the universal representation of enterprise knowledge.

The implementation must keep the domain model independent from:

- PostgreSQL
- SQLAlchemy
- FastAPI
- Redis
- MinIO
- AI Services

The model belongs to the Domain Layer.

---

## Create

backend/core/knowledge/models.py

---

## Technology

Python 3.12+

Pydantic v2

Standard Library

UUID

Datetime

Typing

No external dependencies except Pydantic.

---

## Implement

### KnowledgeObjectHeader

Fields:

- id: UUID
- object_type: str
- organization_id: UUID
- status: KnowledgeObjectStatus
- version: int
- created_at: datetime
- updated_at: datetime

The header is immutable after creation except:

- status
- version
- updated_at

---

### KnowledgeObject

Fields:

- header: KnowledgeObjectHeader
- payload: dict[str, Any]

The payload contains business metadata.

The model shall not contain:

- Relations
- Audit
- Attachments
- Embeddings
- Events
- Permissions

Those belong to dedicated capabilities.

---

### KnowledgeObjectStatus

Create an Enum containing:

- DRAFT
- ACTIVE
- ARCHIVED
- DELETED

---

## Validation

Version >= 1

Payload defaults to empty dictionary.

UUID fields are validated automatically by Pydantic.

---

## Constraints

No business logic.

No persistence.

No repository.

No SQLAlchemy.

No API.

No serialization helpers.

No utility methods except those generated by Pydantic.
